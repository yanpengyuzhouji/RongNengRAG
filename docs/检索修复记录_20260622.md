## 二、修复前测试结果 (基线)

| 指标 | 数值 |
|------|------|
| 域分类准确 | 8/12 (67%) |
| 关键文件命中 | 1/12 (8%) |
| 综合域匹配 | 0/3 (完全失败) |

### 根因分析

**Bug 1 (致命): RRF dense_weight/sparse_weight 是死代码**

`milvus_store.py` 的 `hybrid_search()` 方法接受 `dense_weight`/`sparse_weight` 参数，但内部使用 `RRFRanker(k=60)`，完全忽略传入的权重值。`config.yaml` 中配置的 `dense_weight: 0.5` 和 `sparse_weight: 0.5` 对检索结果无任何影响。

**Bug 2 (高): 域分类用原始子串匹配**

`query_analyzer.py` 的 `_classify_domain()` 使用 `if kw in query` 进行字符级子串匹配:

- "配电装置" 是变电关键词，但包含 "配电" 子串 → 同时命中变电和配电 → 可能误判
- "10kV" 是配电关键词，"10kV变电站" → 同时命中 → 平局释放域过滤 → 跨域噪声
- GB50061 (66kV架空线路) 误标为 "配电" 域

**Bug 3 (高): 元数据加权乘性叠加**

`reranker.py` 的 `_compute_metadata_boost()` 使用乘法叠加:
```
文件名匹配(1.30) × 域匹配(1.05) × 电压匹配(1.10) × 标准类别(1.05) = 1.58x
```
语义相关但不匹配元数据的文档被匹配元数据但语义不相关的文档压制。

## 三、代码修改清单

### 修改1: 域分类 — 子串匹配 → jieba分词+词边界

**文件**: `src/retrieval/query_analyzer.py`

```python
# 修改前 (行131-157)
def _classify_domain(self, query: str) -> Optional[str]:
    for kw in keywords:
        if kw in query:   # 字符级子串匹配
            score += 1

# 修改后
import jieba

def _classify_domain(self, query: str) -> Optional[str]:
    words = set(jieba.lcut(query))  # jieba分词
    for kw in keywords:
        if kw in words:              # 精确词匹配
            score += 1

    # 消歧规则: 变电/配电同时命中时，强变电信号加权
    if "变电" in scores and "配电" in scores:
        strong_biandian = {"变电站", "GIS", "主变", "换流站", ...}
        if any(w in words for w in strong_biandian):
            scores["变电"] += 2

    # 消歧规则: 送电输电强信号加权
    if "送电输电" in scores:
        strong_songdian = {"架空线路", "杆塔", "导线", "输电", ...}
        if any(w in words for w in strong_songdian):
            scores["送电输电"] += 2
```

**效果**: "3~110kV高压配电装置设计" 从误判为配电 → 正确归变电

### 修改3: 元数据加权 — 乘法叠加 → 加法+上限

**文件**: `src/retrieval/reranker.py`

```python
# 修改前 (行232-283)
def _compute_metadata_boost(self, ...) -> float:
    boost = 1.0
    ...
    boost *= 1.30  # 文件名匹配 — 乘法
    boost *= 1.05  # 域匹配
    boost *= 1.10  # 电压匹配
    # 最坏情况: 1.30 × 1.05 × 1.10 × 1.05 = 1.58x
    return boost

# 修改后
def _compute_metadata_boost(self, ...) -> float:
    boost_delta = 0.0
    ...
    boost_delta += 0.10  # 文件名匹配 — 加法 (从1.30乘法降为0.10加法)
    boost_delta += 0.05  # 域匹配
    boost_delta += 0.05  # 电压匹配
    boost_delta += 0.03  # 标准规范
    ...
    # 上限 1.15 (最多提升15%)
    return min(1.15, 1.0 + boost_delta)
```

**效果**: 元数据不再统治语义评分，防止文件名/域标签相似的无关文档冲到顶部
